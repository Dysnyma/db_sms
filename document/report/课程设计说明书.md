## 第一章 概述

### 1.1 项目背景

本课程设计选题为题目二「学生成绩管理系统」。系统通过分析学校日常教学管理中的课程、选课、学生、班级、教师和成绩等核心数据，设计并实现一个集教师授课管理、学生选课管理、成绩管理与统计等功能于一体的小型数据库应用系统。

选题要求使用传统的人工设计方法结合AI辅助工具完成数据库设计的六个阶段：
1. 需求分析
2. 概念结构设计
3. 逻辑结构设计
4. 物理结构设计
5. 数据库实施
6. 数据库运行与维护

## 第二章 需求分析

### 2.1 系统业务需求

根据《数据库系统课程设计》任务书7.2节，学生成绩管理系统需满足以下业务需求：

#### （1）实体与关系

- 一个学生属于一个班级，一个班级有多个学生（**1:N**）
- 一个学生可以选修多门课程，一门课程可以由多个学生选修（**M:N**），通过选课中间表实现
- 一门课程可以由多个教师教授，一个教师可以教授多门课程（**M:N**），通过讲授表和排课表实现

#### （2）选课机制

- 每次选课需要先安排好排课（course_offering），学生再根据排课安排进行选择
- 每门课的排课安排有**有效选课时间**（enroll_start_time ~ enroll_end_time）
- 排课安排有**选课截止时间限制**，达到后不能再选课
- 排课安排有**成绩录入截止时间**，达到后不能再修改成绩
- 选课人数不能超过设置的**上限**（max_students），满员时选课抛出异常
- 学生在选课截止时间前可以**退选**，退选的课程不再显示
- 同一名学生不能选择**同一门课程**中两个不同教师的班级

#### （3）成绩管理

- 教师对其所授课程的选课学生进行成绩录入
- 成绩录入后**触发器自动实时计算**学生的学籍分和绩点分（GPA）
- 学籍分 = Σ(已选课程成绩 × 课程学分) / Σ(已选全部课程学分)
- 绩点分 = Σ(课程学分 × 对应绩点) / Σ(全部课程学分)
- 成绩录入既可以**逐条录入**也可以**批量CSV录入**
- 成绩录入截止时间后不能再修改

#### （4）统计报表

所有统计报表要求使用**存储过程**实现，向应用程序传递数据：

- 按班级排序输出学生学籍分（sp_student_roster）
- 按班级+课程统计平均分、最高分、最低分、及格率（sp_class_grade_report）
- 统计某学生某学期的所有课程平均成绩（sp_student_semester_avg）
- 教师授课统计（sp_teacher_info / sp_teacher_list）

#### （5）角色权限

设计**三个角色**，分别对应不同系统功能：

| 角色 | 功能 |
|------|------|
| 学生 | 查可选课程、选课、退选、查自己成绩和学籍分、查学期均分 |
| 教师 | 逐条/批量录入成绩、查看所授课程学生名单 |
| 教务管理员 | 数据概览、基础数据CRUD、选课安排CRUD、选课管理、统计报表、备份恢复 |

#### （6）数据维护

- 学生、班级、课程、教师等基础数据的**增删改查**
- 全部使用**逻辑删除**（is_deleted字段），保留数据痕迹
- 学生基本信息获取要求使用**视图**
- 学籍分在增删改时要求使用**触发器**自动计算
- 选课安排维护注意有效时间、人数限制等字段设定
- 选课人数使用**触发器**实现实时动态更新
- 数据库备份和恢复功能

### 2.2 数据流图（DFD）

本系统采用自顶向下的结构化分析方法。

#### 顶层数据流图（系统级）

```
                    ┌─────────────────┐
      学号/工号/admin │                │ 查询结果
          选课申请  ──│  学生成绩管理   │──→ 统计报表
          成绩录入 ──→│     系统       │──→ 备份文件
         CRUD操作  ──│                │──→ 学籍分/绩点分
                    └─────────────────┘
      学生/教师/教务                   学生/教师/教务
```

#### 0层数据流图（功能级）

**P1 基础数据管理：**

```
教务 ──→ (1.1 班级管理) ──→ D1 班级表
教务 ──→ (1.2 学生管理) ──→ D2 学生表
教务 ──→ (1.3 课程管理) ──→ D3 课程表
教务 ──→ (1.4 教师管理) ──→ D4 教师表
教务 ──→ (1.5 讲授关系管理) ──→ D7 讲授表
```

**P2 排课管理：**

```
教务 ──→ (2.1 排课CRUD) ──→ D5 排课表 (course_offering)
                            ↑
                            └── 关联 D3 课程表 + D4 教师表 + D7 讲授表
```

**P3 选课管理：**

```
学生 ──→ (3.1 查看可选课程) ──→ 视图 v_course_plan ──→ 返回排课列表
学生 ──→ (3.2 选课 sp_enroll) ──→ D6 选课表 ──→ 触发器更新 D5 排课人数
学生 ──→ (3.3 退选 sp_unenroll) ──→ D6 选课表 ──→ 触发器更新 D5 排课人数
```

**P4 成绩管理：**

```
教师 ──→ (4.1 录入成绩 sp_grade_input) ──→ D6 选课表 ──→ 触发器更新 D2 学籍分/绩点分
教师 ──→ (4.2 批量CSV录入) ──→ 循环调用 sp_grade_input ──→ 同上
```

**P5 查询统计：**

```
学生/教师/教务 ──→ (5.1 统计查询) ──→ 存储过程 ──→ 返回结果集
    ├── sp_student_roster (班级名单)
    ├── sp_class_grade_report (成绩统计)
    ├── sp_student_semester_avg (学期均分)
    ├── sp_teacher_info (教师信息)
    ├── sp_teacher_list (教师列表)
    └── v_student_message / v_enrollment (视图查询)
```

**P6 备份恢复：**

```
教务 ──→ (6.1 备份) ──→ mysqldump ──→ D8 备份文件(.sql)
教务 ──→ (6.2 恢复) ←── mysql < D8 备份文件 ──→ 重建数据库
```

#### 2.3.5 处理过程

处理过程描述系统对数据的加工逻辑。

##### 处理过程1：基础数据管理

| 属性 | 值 |
|------|-----|
| 处理过程名 | 基础数据管理 |
| 说明 | 对班级、学生、课程、教师、讲授关系五类基础数据的增删改查操作 |
| 输入 | 教务管理员的CRUD操作指令 |
| 输出 | 更新后的数据库记录 |
| 处理 | 1. 查询：从对应表中SELECT数据并展示 2. 新增：校验必填字段和外键约束后INSERT 3. 修改：逐字段确认后UPDATE（回车保留原值）4. 删除：确认后SET is_deleted=1逻辑删除 |

##### 处理过程2：排课管理

| 属性 | 值 |
|------|-----|
| 处理过程名 | 排课管理 |
| 说明 | 教务管理员创建和管理每次选课的排课安排 |
| 输入 | 课程ID、教师ID、学期、选课上限、三个时间节点 |
| 输出 | 写入D5排课表的新记录 |
| 处理 | 1. 选择课程 → 2. 系统查询D7讲授表，列出有资格教授该课的教师 → 3. 选择教师 → 4. 填写学期/上限/选课开始/选课截止/成绩截止 → 5. INSERT |

##### 处理过程3：选课处理（sp_enroll）

| 属性 | 值 |
|------|-----|
| 处理过程名 | 选课处理（sp_enroll） |
| 说明 | 学生在选课有效期内选择某排课 |
| 输入 | 学号（p_student_no）、排课ID（p_plan_id） |
| 输出 | 新增D6选课记录，触发器更新D5排课当前人数+1 |
| 处理 | 1. 根据学号查学生ID（调用fn_get_student_id）→ 2. 校验排课存在且在选课期内（NOW BETWEEN enroll_start AND enroll_end）→ 3. 校验未选过相同排课 → 4. 校验未选过同一课程的其他教师 → 5. INSERT INTO enrollment → 6. 触发器trg_enrollment_before_insert检查名额 → 7. 触发器trg_enrollment_after_insert更新current_students +1 → 8. COMMIT（事务包裹，异常时ROLLBACK） |

##### 处理过程4：退选处理（sp_unenroll）

| 属性 | 值 |
|------|-----|
| 处理过程名 | 退选处理（sp_unenroll） |
| 说明 | 学生在选课截止前取消已选的课程 |
| 输入 | 学号（p_student_no）、排课ID（p_plan_id） |
| 输出 | D6选课记录is_deleted=1，触发器更新D5排课当前人数-1 |
| 处理 | 1. 根据学号查学生ID → 2. 校验已选该课且未退选 → 3. 校验仍在选课截止时间内（NOW <= enroll_end_time）→ 4. UPDATE enrollment SET is_deleted=1 → 5. 触发器trg_enrollment_after_update检测到0→1转换 → 6. 更新current_students -1 → 7. COMMIT |

##### 处理过程5：成绩录入（sp_grade_input）

| 属性 | 值 |
|------|-----|
| 处理过程名 | 成绩录入（sp_grade_input） |
| 说明 | 教师对其授课的选课学生进行成绩录入 |
| 输入 | 教师工号（p_teacher_no）、排课ID（p_plan_id）、学生学号（p_student_no）、成绩（p_score） |
| 输出 | 更新D6选课表成绩字段，触发器自动重算D2学籍分和绩点分 |
| 处理 | 1. 根据工号查教师ID（调用fn_get_teacher_id）→ 2. 校验教师是该排课授课教师且在成绩录入截止时间内 → 3. 根据学号查学生ID（调用fn_get_student_id）→ 4. 校验学生已选该课 → 5. 校验0 ≤ p_score ≤ 100 → 6. UPDATE enrollment SET score=p_score → 7. 触发器trg_enrollment_after_update_score检测成绩变化 → 8. 重算weighted_score = Σ(score×credit)/Σ(credit) → 9. 重算gpa = Σ(credit×point)/Σ(credit) → 10. COMMIT |

##### 处理过程6：查询可选课程（sp_show_courses）

| 属性 | 值 |
|------|-----|
| 处理过程名 | 查询可选课程（sp_show_courses） |
| 说明 | 学生查询当前可选的课程排课列表 |
| 输入 | 学号（p_student_no） |

## 第三章 概念结构设计

### 3.1 全局ER图

全局ER图包含8个实体/关系。主要关系如下：

#### 实体清单

1. **班级 (class)** — 学生的归属单位
2. **学生 (student)** — 选课和成绩的主体
3. **课程 (course)** — 教学课程
4. **教师 (teacher)** — 授课教师
5. **排课 (course_offering)** — 课程与教师的排课安排

#### 关系清单

6. **选课 (enrollment)** — 学生与排课的M:N关系
7. **讲授 (teacher_course)** — 教师与课程的M:N关系
8. **绩点规则 (grade_point_rule)** — 成绩→绩点映射

#### 实体间关系

```
  class ──1:N── student
  student ──M:N── course_offering ──N:1── course
                                   ──N:1── teacher
  teacher ──M:N── course (通过 teacher_course)
  student.gpa ←── grade_point_rule (计算时引用)
```

#### 关键设计决策

1. **学生与班级：1:N** — student表加class_id外键，不建中间表
2. **学生与课程：M:N** — 通过enrollment中间表实现，中间表同时记录成绩
3. **课程与教师：M:N** — 通过teacher_course（讲授资格）和course_offering（实际排课）两层实现
4. **全部逻辑删除** — is_deleted字段管数据状态，status管业务状态
5. **学籍分和绩点分** — 由触发器自动维护，不通过应用程序计算

注：完整ER图见附件 `images/总体ER图.png`。

### 4.2 规范化设计说明

1. **第一范式（1NF）**：所有字段均为原子值，无重复组。例如student表中的字段均为单一值，不存在一行中包含多个学号的情况。

2. **第二范式（2NF）**：每个非主键字段完全依赖于主键。enrollment表的score字段同时依赖于offering_id和student_id（联合主键逻辑），不存在部分函数依赖。weighted_score和gpa仅依赖于student.id。

3. **第三范式（3NF）**：不存在传递依赖。student表通过class_id关联class表获取班级信息，而非在学生表中冗余存储班级名称。enrollment通过offering_id关联course_offering，再关联course和teacher获取课程名和教师名。

4. **反范式化设计**：未采用字段冗余。在高频查询中（如v_enrollment视图需要显示学生姓名、课程名、教师名），通过JOIN多表而非存储冗余字段，保证了数据一致性。

5. **约束完整性**：
   - 所有外键关系均定义了FOREIGN KEY约束
   - 学号、工号、课程名等关键字段均定义了UNIQUE索引
   - 选课人数通过触发器维护CHECK逻辑（max_students >= current_students）
   - 所有表采用逻辑删除，保护数据完整性

### 4.4 视图设计

#### 视图1：v_student_message（学生基本信息视图）

**用途：** 向学生和教务提供基本信息，不暴露学籍分/绩点分敏感数据

```sql
CREATE VIEW v_student_message AS
SELECT s.id AS student_id, s.name AS student_name, s.no AS student_no,
       s.class_id, c.name AS class_name, c.grade, c.major, s.status
FROM student s
INNER JOIN class c ON s.class_id = c.id
```

#### 视图2：v_course_plan（选课安排列表视图）

**用途：** 学生选课时看到的排课列表，直接输出课程名+教师名+选课状态

```sql
CREATE VIEW v_course_plan AS
SELECT co.id AS plan_id, co.course_id, c.name AS course_name, c.credit,
       co.teacher_id, t.name AS teacher_name, co.semester,
       co.max_students, co.current_students,
       co.max_students - co.current_students AS remaining_seats,
       co.enroll_start_time, co.enroll_end_time, co.grade_deadline,
       CASE WHEN NOW() < co.enroll_start_time THEN '未开始'
            WHEN NOW() BETWEEN co.enroll_start_time AND co.enroll_end_time THEN '选课中'
            WHEN NOW() > co.grade_deadline THEN '已结束'
            ELSE '已截止' END AS enroll_status,
       co.status
FROM course_offering co
INNER JOIN course c ON co.course_id = c.id
INNER JOIN teacher t ON co.teacher_id = t.id
WHERE co.is_deleted = 0;
```

```sql
SELECT e.id AS enroll_id, e.offering_id, e.student_id,
       s.name AS student_name, s.no AS student_no,
       c.id AS course_id, c.name AS course_name, c.credit,
       t.id AS teacher_id, t.name AS teacher_name,
       co.semester, e.score, e.create_time AS enroll_time
FROM enrollment e
INNER JOIN student s ON e.student_id = s.id
INNER JOIN course_offering co ON e.offering_id = co.id
INNER JOIN course c ON co.course_id = c.id
INNER JOIN teacher t ON co.teacher_id = t.id
WHERE e.is_deleted = 0;
```

### 4.5 存储过程设计

| 编号 | 存储过程 | 参数 | 功能 | 特殊逻辑 |
|------|----------|------|------|----------|
| SP1 | sp_show_courses | p_student_no VARCHAR(20) | 查询可选课程 | 过滤已选同课程的所有教师、过滤选课期外、过滤满员 |
| SP2 | sp_enroll | p_student_no, p_plan_id | 选课 | 4步校验 + 事务 + 触发器自动检查名额/更新人数 |

| SP4 | sp_grade_input | p_teacher_no, p_plan_id, p_student_no, p_score | 录入成绩 | 三重身份校验 + 时间校验 + 成绩范围校验 + 触发器自动重算学籍分/绩点分 |

### 4.6 触发器设计

| 编号 | 触发器 | 触发时机 | 功能 |
|------|--------|----------|------|
| TRG1 | trg_enrollment_before_insert | BEFORE INSERT ON enrollment | 选课前检查名额：current_students >= max_students时SIGNAL异常 |
| TRG2 | trg_enrollment_after_insert | AFTER INSERT ON enrollment | 选课后更新course_offering.current_students +1 |
| TRG3 | trg_enrollment_after_update | AFTER UPDATE ON enrollment | 检测is_deleted 0→1转换时current_students -1 |
| TRG4 | trg_enrollment_after_insert_score | AFTER INSERT ON enrollment | 选课时如有成绩则自动重算该学生的学籍分和绩点分 |
| TRG5 | trg_enrollment_after_update_score | AFTER UPDATE ON enrollment | 成绩变化（<=>比较NULL安全）后自动重算学籍分和绩点分 |

**触发器的学籍分计算公式：**

```sql
weighted_score = ROUND(SUM(e.score * c.credit) / SUM(c.credit), 2)
```

**触发器的绩点分计算公式（GPA）：**

```sql
gpa = ROUND(SUM(c.credit * r.point) / SUM(c.credit), 2)
-- 其中 r.point 来自 grade_point_rule 表，
-- 通过 e.score BETWEEN r.min_score AND r.max_score 匹配
```

### 5.3 AI辅助物理设计记录

#### 5.3.1 AI的索引建议

AI分析了以下高频查询模式并给出索引建议：

1. **按学号查学生**（登录、选课、成绩录入时频繁） → uk_student_no唯一索引
2. **按班级查学生名单**（roster报表） → idx_student_class普通索引
3. **按排课查选课学生**（教师查看名单） → (offering_id, student_id)复合唯一索引
4. **按学生查选课记录**（成绩单、GPA计算） → idx_enr_student普通索引
5. **按教师查排课**（教师端所有功能） → idx_co_teacher普通索引

#### 5.3.2 EXPLAIN验证

对关键查询使用EXPLAIN分析执行计划：

```sql
-- 查询：某班级学生名单
EXPLAIN SELECT * FROM student WHERE class_id = 1 AND is_deleted = 0;
-- 结果：type=ref, key=idx_student_class, rows≈30
-- 分析：使用idx_student_class索引，扫描行数少，效率高

-- 查询：某学生已选课程
EXPLAIN SELECT * FROM enrollment WHERE student_id = 5 AND is_deleted = 0;
-- 结果：type=ref, key=idx_enr_student, rows≈8
-- 分析：使用idx_enr_student索引，效率高

-- 查询：某排课的所有学生
EXPLAIN SELECT * FROM enrollment WHERE offering_id = 10 AND is_deleted = 0;
-- 结果：type=ref, key=uk_enrollment, rows≈20
-- 分析：使用(offering_id, student_id)索引前缀，效率高
```

#### 5.3.3 最终采纳方案

经EXPLAIN验证后采纳了全部索引建议。覆盖所有WHERE条件字段，type均为ref或const级别，无全表扫描。未采纳AI建议的任何索引均未发现对查询性能有帮助。

## 第八章 系统运行与维护

### 8.1 运维变更：学籍分→学位分（GPA）

按照任务书7.2.2节要求，实现数据独立性运维变更。

**变更需求：** 学校将「学籍分」的概念变更为「学位分」，即GPA（Grade Point Average）。

**变更前公式：**
```
学籍分 = Σ(成绩 × 学分) / Σ(学分)
```

**变更后公式（GPA）：**
```
学位分(GPA) = Σ(学分 × 绩点) / Σ(学分)
其中绩点由成绩区间映射：0-59→0, 60-69→1, 70-79→2, 80-89→3, 90-100→4
```

**变更实现方式：**

1. **不修改任何应用层代码（CLI和Streamlit）**——Python代码中对 `gpa` 字段的读取保持不变
2. **不修改存储过程**——sp_student_roster等报表SP仍查询 `student.gpa` 字段
3. **只修改数据库触发器** trg_enrollment_after_update_score 中的更新逻辑，将gpa字段从旧的学籍分公式改为GPA公式
4. 若需要回退，只需恢复旧的触发器代码

**数据独立性体现：**

- 应用程序通过字段名 `gpa` 访问数据，不关心该值是如何计算的
- 计算逻辑的变更完全在数据库层面完成
- 前端UI无需任何修改——原先显示「绩点分」的metric卡片继续正常显示

- AI对某些UI框架细节存在知识盲区，需要人工验证

#### 9.4.2 AI的亮点与局限

**亮点：**
- 能够快速理解复杂的业务需求（26条功能点），生成结构正确的SQL和Python代码框架

1. **AI是强大的辅助工具，但不能替代人的架构设计。** 核心的设计决策（如实体关系建模、范式设计、索引策略）必须由人工主导，AI仅提供建议供参考。

#### A.2 关键存储过程——选课（sp_enroll）

```sql
CREATE PROCEDURE sp_enroll(p_student_no VARCHAR(20), p_plan_id INT)
BEGIN
    DECLARE v_student_id INT;
    DECLARE v_msg VARCHAR(200);
    DECLARE EXIT HANDLER FOR SQLEXCEPTION
    BEGIN
        ROLLBACK;
        RESIGNAL;
    END;
    START TRANSACTION;
    
    -- 1. 根据学号找学生ID
    SET v_student_id = fn_get_student_id(p_student_no);
    IF v_student_id IS NULL THEN
        SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '学生不存在';
    END IF;
    
    -- 2. 检查排课是否存在且在选课期内
    IF NOT EXISTS (
        SELECT 1 FROM course_offering
        WHERE id = p_plan_id AND is_deleted = 0
          AND NOW() BETWEEN enroll_start_time AND enroll_end_time
    ) THEN SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '该课程不在选课期内或不存在'; END IF;
    
    -- 3. 检查是否已选过同一排课
    IF EXISTS (SELECT 1 FROM enrollment
        WHERE offering_id = p_plan_id AND student_id = v_student_id AND is_deleted = 0
    ) THEN SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '你已经选过这门课了'; END IF;
    
    -- 4. 检查是否已选过同一门课的其他老师
    IF EXISTS (SELECT 1 FROM enrollment e
        JOIN course_offering co ON e.offering_id = co.id
        WHERE e.student_id = v_student_id AND e.is_deleted = 0
          AND co.course_id = (SELECT course_id FROM course_offering WHERE id = p_plan_id)
    ) THEN SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '你已选过该课程的其他教师'; END IF;
    
    -- 5. 选课（触发器自动检查名额+更新人数）
    INSERT INTO enrollment (offering_id, student_id) VALUES (p_plan_id, v_student_id);
    COMMIT;
    SELECT '选课成功' AS message;
END;
```

#### A.3 关键触发器——绩点自动计算（trg_enrollment_after_update_score）

```sql
CREATE TRIGGER trg_enrollment_after_update_score
AFTER UPDATE ON enrollment FOR EACH ROW
BEGIN
    IF NOT (OLD.score <=> NEW.score) THEN
        -- 更新学籍分
        UPDATE student s SET weighted_score = (
            SELECT ROUND(SUM(e.score * c.credit) / SUM(c.credit), 2)
            FROM enrollment e
            JOIN course_offering co ON e.offering_id = co.id
            JOIN course c ON co.course_id = c.id
            WHERE e.student_id = NEW.student_id
              AND e.score IS NOT NULL AND e.is_deleted = 0
        ) WHERE s.id = NEW.student_id;
        
        -- 更新绩点分(GPA)
        UPDATE student s SET gpa = (
            SELECT ROUND(SUM(c.credit * r.point) / SUM(c.credit), 2)
            FROM enrollment e
            JOIN course_offering co ON e.offering_id = co.id
            JOIN course c ON co.course_id = c.id
            JOIN grade_point_rule r ON e.score BETWEEN r.min_score AND r.max_score
            WHERE e.student_id = NEW.student_id
              AND e.score IS NOT NULL AND e.is_deleted = 0
        ) WHERE s.id = NEW.student_id;
    END IF;
END;
```

#### B.2 分页器类（Paginator）

```python
class Paginator:
    """分页器：只管切片和翻页，不管显示"""
    def __init__(self, rows, per_page=10):
        self.rows = rows; self.per_page = per_page; self.page = 0
    
    @property
    def items(self):
        start = self.page * self.per_page
        return self.rows[start:start + self.per_page]
    
    @property
    def total(self):
        return (len(self.rows) + self.per_page - 1) // self.per_page
    
    @property
    def info(self):
        return f'第 {self.page+1}/{self.total} 页，共 {len(self.rows)} 条'
    
    def next(self):
        if self.page < self.total - 1: self.page += 1
    
    def prev(self):
        if self.page > 0: self.page -= 1
    
    def reset(self):
        self.page = 0
```
