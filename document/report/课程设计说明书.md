## 第二章 需求分析

### 2.1 系统业务需求

根据任务书要求，学生成绩管理系统需满足以下业务需求：

**（1）实体与关系**
- 一个学生属于一个班级，一个班级有多个学生（**1:N**）
- 一个学生可以选修多门课程，一门课程可以由多个学生选修（**M:N**），通过选课中间表实现
- 一门课程可以由多个教师教授，一个教师可以教授多门课程（**M:N**），通过讲授表和排课表实现

**（2）选课机制**
- 排课安排有有效选课时间、选课截止时间和成绩录入截止时间
- 选课人数不能超过上限（max_students），满员时选课抛出异常
- 同一名学生不能选择同一门课程中两个不同教师的班级
- 学生在选课截止时间前可以退选

**（3）成绩管理**
- 教师对其所授课程的选课学生进行成绩录入
- 成绩录入后触发器自动实时计算学籍分和绩点分（GPA）
- 成绩录入支持逐条录入和批量CSV录入

**（4）统计报表**
- 按班级排序输出学生学籍分（sp_student_roster）
- 按班级+课程统计平均分/最高分/最低分/及格率（sp_class_grade_report）
- 统计某学生某学期均分（sp_student_semester_avg）
- 教师授课统计（sp_teacher_info / sp_teacher_list）

**（5）角色权限**
- 学生：查可选课程、选课、退选、查成绩和学籍分、学期均分
- 教师：逐条/批量录入成绩、查看所授课程学生名单
- 教务管理员：数据概览、基础数据CRUD、排课管理、统计报表、备份恢复

**（6）数据维护**
- 全部采用逻辑删除（is_deleted字段）
- 学籍分和选课人数使用触发器自动计算
- 数据库备份和恢复功能

## 第三章 概念结构设计

### 3.1 全局ER图

#### 实体清单

1. **班级（class）** — 学生的归属单位
2. **学生（student）** — 选课和成绩的主体
3. **课程（course）** — 教学课程
4. **教师（teacher）** — 授课教师
5. **排课（course_offering）** — 课程与教师的排课安排

#### 关系清单

6. **选课（enrollment）** — 学生与排课的M:N关系
7. **讲授（teacher_course）** — 教师与课程的M:N关系
8. **绩点规则（grade_point_rule）** — 成绩→绩点映射

#### 实体间关系

```
class ──1:N── student
student ──M:N── course_offering ──N:1── course
                                 ──N:1── teacher
teacher ──M:N── course（通过 teacher_course）
```

> [截图：总体ER图]

#### 关键设计决策

1. **学生与班级：1:N** — student表加class_id外键，不建中间表
2. **学生与课程：M:N** — 通过enrollment中间表实现，同时记录成绩
3. **课程与教师：M:N** — 通过teacher_course（讲授资格）和course_offering（实际排课）两层实现
4. **全部逻辑删除** — is_deleted管数据状态，status管业务状态
5. **学籍分和绩点分** — 由触发器自动维护，不通过应用程序计算

### 3.2 AI辅助概念设计记录

#### 3.2.1 使用AI工具辅助ER图设计

AI帮助明确了实体关系的建模决策：确认学生-班级是1:N关系不需中间表；建议使用teacher_course和course_offering双层设计。

#### 3.2.2 人工干预记录

| AI生成 | 人工修改 | 理由 |
|--------|----------|------|
| course_offering含effective_time | 删除 | 与enroll_start_time语义重复 |
| enrollment同时有enroll_time和create_time | 删除enroll_time | create_time就是选课时间 |
| 遗漏绩点规则表 | 新增grade_point_rule | GPA计算需要成绩-绩点映射 |

### 4.2 规范化设计说明

1. **1NF**：所有字段均为原子值，无重复组
2. **2NF**：每个非主键字段完全依赖于主键，无部分函数依赖
3. **3NF**：不存在传递依赖（通过class_id关联class表获取班级信息，不冗余存储班级名）
4. **约束完整性**：全部外键定义了FOREIGN KEY约束，学号工号定义了UNIQUE索引

### 4.4 视图设计

#### 视图1：v_student_message（学生基本信息视图）

```sql
CREATE VIEW v_student_message AS
SELECT s.id AS student_id, s.name AS student_name, s.no AS student_no,
       s.class_id, c.name AS class_name, c.grade, c.major, s.status
FROM student s INNER JOIN class c ON s.class_id = c.id
WHERE s.is_deleted = 0;
```

#### 视图2：v_course_plan（选课安排列表视图）

```sql
CREATE VIEW v_course_plan AS
SELECT co.id AS plan_id, co.course_id, c.name AS course_name, c.credit,
       co.teacher_id, t.name AS teacher_name, co.semester,
       co.max_students, co.current_students,
       co.max_students - co.current_students AS remaining_seats,
       co.enroll_start_time, co.enroll_end_time, co.grade_deadline,
       CASE WHEN NOW() < co.enroll_start_time THEN '未开始'
            WHEN NOW() BETWEEN co.enroll_start_time AND co.enroll_end_time THEN '选课中'
            WHEN NOW() > co.grade_deadline THEN '已结束'
            ELSE '已截止' END AS enroll_status,
       co.status
FROM course_offering co
INNER JOIN course c ON co.course_id = c.id
INNER JOIN teacher t ON co.teacher_id = t.id
WHERE co.is_deleted = 0;
```

#### 视图3：v_enrollment（选课详情视图）

```sql
CREATE VIEW v_enrollment AS
SELECT e.id AS enroll_id, e.offering_id, e.student_id,
       s.name AS student_name, s.no AS student_no,
       c.id AS course_id, c.name AS course_name, c.credit,
       t.id AS teacher_id, t.name AS teacher_name,
       co.semester, e.score, e.create_time AS enroll_time
FROM enrollment e
INNER JOIN student s ON e.student_id = s.id
INNER JOIN course_offering co ON e.offering_id = co.id
INNER JOIN course c ON co.course_id = c.id
INNER JOIN teacher t ON co.teacher_id = t.id
WHERE e.is_deleted = 0;
```

### 4.6 触发器设计

| 编号 | 触发器 | 触发时机 | 功能 |
|------|--------|----------|------|
| TRG1 | trg_enrollment_before_insert | BEFORE INSERT | 选课前检查名额 |
| TRG2 | trg_enrollment_after_insert | AFTER INSERT | 选课后current_students+1 |
| TRG3 | trg_enrollment_after_update | AFTER UPDATE | 退选时current_students-1 |
| TRG4 | trg_enrollment_after_insert_score | AFTER INSERT | 选课带成绩时重算学籍分/绩点分 |
| TRG5 | trg_enrollment_after_update_score | AFTER UPDATE | 成绩变化时重算学籍分/绩点分 |

学籍分公式：weighted_score = Σ(score × credit) / Σ(credit)

绩点分公式：gpa = Σ(credit × point) / Σ(credit)

## 第五章 物理设计

### 5.1 存储结构

| 配置项 | 值 | 理由 |
|--------|-----|------|
| 存储引擎 | InnoDB | 支持事务、行级锁、外键约束 |
| 字符集 | utf8mb4 | 完全支持中文和4字节Unicode字符 |
| 排序规则 | utf8mb4_unicode_ci | Unicode标准排序，兼容性好 |
| 主键策略 | INT AUTO_INCREMENT | 4字节整型，查询效率高 |

### 5.2 索引设计

| 表名 | 索引字段 | 类型 | 设计理由 |
|------|----------|------|----------|
| class | name, grade, major | 普通索引 | 按班级名/年级/专业查询 |
| student | no | 唯一索引 | 学号唯一性约束，最频繁查询条件 |
| student | class_id | 普通索引 | 按班级查询学生名单（高频） |
| course | name | 唯一索引 | 课程名唯一性约束 |
| teacher | no | 唯一索引 | 工号唯一性约束 |
| course_offering | teacher_id | 普通索引 | 按教师查排课（高频） |
| course_offering | semester | 普通索引 | 按学期筛选 |
| course_offering | (enroll_start_time, enroll_end_time) | 复合索引 | 选课时间范围查询 |

### 8.1 运维变更示例

按照任务书要求，实现数据独立性运维变更。

**变更需求：** 学校将「学籍分」的概念变更为「学位分」，即GPA。

#### A.1 关键存储过程——选课（sp_enroll）

```sql
CREATE PROCEDURE sp_enroll(p_student_no VARCHAR(20), p_plan_id INT)
BEGIN
    DECLARE v_student_id INT;
    DECLARE EXIT HANDLER FOR SQLEXCEPTION
    BEGIN ROLLBACK; RESIGNAL; END;
    START TRANSACTION;
    SET v_student_id = fn_get_student_id(p_student_no);
    IF v_student_id IS NULL THEN
        SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '学生不存在'; END IF;
    IF NOT EXISTS (SELECT 1 FROM course_offering WHERE id = p_plan_id
        AND is_deleted = 0 AND NOW() BETWEEN enroll_start_time AND enroll_end_time
    ) THEN SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '该课程不在选课期内或不存在'; END IF;
    IF EXISTS (SELECT 1 FROM enrollment WHERE offering_id = p_plan_id
        AND student_id = v_student_id AND is_deleted = 0
    ) THEN SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '你已经选过这门课了'; END IF;
    IF EXISTS (SELECT 1 FROM enrollment e JOIN course_offering co ON e.offering_id = co.id
        WHERE e.student_id = v_student_id AND e.is_deleted = 0
        AND co.course_id = (SELECT course_id FROM course_offering WHERE id = p_plan_id)
    ) THEN SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '你已选过该课程的其他教师'; END IF;
    INSERT INTO enrollment (offering_id, student_id) VALUES (p_plan_id, v_student_id);
    COMMIT;
    SELECT '选课成功' AS message;
END;
```

#### A.2 关键触发器——绩点自动计算

```sql
CREATE TRIGGER trg_enrollment_after_update_score
AFTER UPDATE ON enrollment FOR EACH ROW
BEGIN
    IF NOT (OLD.score <=> NEW.score) THEN
        UPDATE student s SET weighted_score = (
            SELECT ROUND(SUM(e.score * c.credit) / SUM(c.credit), 2)
            FROM enrollment e JOIN course_offering co ON e.offering_id = co.id
            JOIN course c ON co.course_id = c.id
            WHERE e.student_id = NEW.student_id AND e.score IS NOT NULL AND e.is_deleted = 0
        ) WHERE s.id = NEW.student_id;
        UPDATE student s SET gpa = (
            SELECT ROUND(SUM(c.credit * r.point) / SUM(c.credit), 2)
            FROM enrollment e JOIN course_offering co ON e.offering_id = co.id
            JOIN course c ON co.course_id = c.id
            JOIN grade_point_rule r ON e.score BETWEEN r.min_score AND r.max_score
            WHERE e.student_id = NEW.student_id AND e.score IS NOT NULL AND e.is_deleted = 0
        ) WHERE s.id = NEW.student_id;
    END IF;
END;
```
