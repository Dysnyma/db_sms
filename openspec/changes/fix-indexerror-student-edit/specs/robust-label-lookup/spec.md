## ADDED Requirements

### Requirement: 标签 ID 安全反向查找

当通过 ID 反向查找标签时，系统必须安全处理目标 ID 不存在于映射中的情况。

#### Scenario: 正常查找——ID 存在于映射中

- **WHEN** 调用反向查找，传入一个存在于映射中的 ID
- **THEN** 系统必须返回对应的标签值

#### Scenario: 异常查找——ID 不存在于映射中

- **WHEN** 调用反向查找，传入一个不存在于映射中的 ID
- **THEN** 系统不得抛出 `IndexError` 或任何未捕获异常
- **THEN** 系统必须返回一个安全的默认值

#### Scenario: 编辑页面——班级已被删除

- **WHEN** 在「修改」模式下选择一名学生
- **AND** 该学生的 `class_id` 对应的班级已被逻辑删除（`is_deleted=1`）
- **THEN** 页面不得崩溃
- **THEN** 编辑表单必须正常渲染，班级下拉框默认选中第一个可用班级
- **THEN** 页面必须在班级选择框上方显示提示文字："原班级已删除，请重新选择"

#### Scenario: 编辑页面——正常班级

- **WHEN** 在「修改」模式下选择一名学生
- **AND** 该学生的班级在 `clmap` 中正常存在
- **THEN** 行为必须与修复前一致：下拉框默认选中该学生原班级
