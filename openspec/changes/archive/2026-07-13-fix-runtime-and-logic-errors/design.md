## Context

当前 `src/` 目录下的 4 个文件存在经 Pylint 和 Flake8 验证的 7 处代码缺陷，分别为：

- **E0606（2 处）**：变量在条件分支中可能赋值前使用，属于真实运行时风险
- **F541（4 处）**：f-string 中无任何 `{}` 占位符，纯属冗余语法
- **F401（2 处）**：导入的模块名称从未使用，浪费解析时间
- **E712（1 处）**：`== False` 比较应使用 `is False` 或 `not` 运算符

所有问题均为局部、无副作用的代码级修复，不涉及架构变更。

## Goals / Non-Goals

**Goals:**
- 消除所有 7 处 Pylint E 类 / Flake8 F/E 类错误
- `flake8 src/ --max-line-length=120` 零通过
- 修复后的两个 E0606 变量路径各自都有兜底默认值

**Non-Goals:**
- 不引入新功能或新依赖
- 不改变函数签名、返回值语义或 UI 行为
- 不修复 Pylint Warning / Convention / Refactor 类问题

## Decisions

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| **E0606 兜底值** | `None` / 空字符串 / 空列表 | 根据变量类型选择最安全的值 | `pages` 兜底为空字典（`{}`），`course_name` 兜底为空字符串（`""`），仅在无法正常赋值时生效 |
| **== False 修复** | `is False` / `not` | `not` | `paged` 是布尔参数，`if not paged` 是最 Pythonic 且与上游调用约定一致的写法 |
| **F401 导入删除** | 保留注释 / 直接删除 | 直接删除 | 未使用的导入无保留价值，注释只会增加维护噪音 |

## Risks / Trade-offs

- **[风险] E0606—app.py**: 兜底 `pages = {}` 为空字典，若角色异常进入 `/` 路径，`st.navigation({})` 会清空侧边栏，用户无法操作。→ **当前已由外层 `if not st.session_state.user` 守卫**，不会到达兜底路径；兜底仅用于满足静态分析契约。
- **[风险] E0606—teacher.py**: `course_name` 兜底为 `""` 后，会在第 82 行打印空字符串，而非崩溃。→ 用户看到一个空名称提示，可接受。此路径仅在输入非数字时触发，属罕见边界。
- **[低风险] F541 改动**：仅移除 `f` 前缀，字符串字面量不变，零影响。
- **[低风险] F401 改动**：函数 `input_choice` 和 `pause` 在 `teacher.py` 中无引用，删除后不受影响。
